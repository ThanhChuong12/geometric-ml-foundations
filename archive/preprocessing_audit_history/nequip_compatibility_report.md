# NequIP Compatibility Report

This report evaluates the compatibility of the preprocessed datasets generated by `scripts/preprocess.py` with the requirements of the **NequIP** (Neural Equivariant Interatomic Potentials) framework.

---

## 📋 1. Current Output Schema vs. Expected NequIP Schema

The tables below contrast the fields currently serialized in the `.pt` datasets against those expected by standard NequIP configuration and training pipelines.

### Current Output Schema
The `.pt` files contain serialized objects (either dictionaries or PyTorch Geometric `Data` objects) with the following fields:

| Field Key | Type | Shape | Description |
|---|---|---|---|
| `z` | `torch.long` | `[N_atoms]` | Atomic numbers |
| `pos` | `torch.float32` | `[N_atoms, 3]` | Coordinates (centered) |
| `edge_index` | `torch.long` | `[2, N_edges]` | Directed edge indices |
| `y` | `torch.float32` | `[1]` | Total energy (normalized) |
| `forces` | `torch.float32` | `[N_atoms, 3]` | Atomic forces (normalized) |

---

### Expected NequIP Schema (`AtomicDataDict`)
The NequIP framework identifies variables using the `AtomicDataDict` constants:

| Key Constant | Standard String | Type | Required / Optional | Description |
|---|---|---|---|---|
| `POSITIONS_KEY` | `"pos"` | `torch.float32` | **Required** | Coordinates of atoms |
| `ATOMIC_NUMBERS_KEY` | `"atomic_numbers"` | `torch.long` | **Required** | Chemical species |
| `TOTAL_ENERGY_KEY` | `"total_energy"` | `torch.float32` | **Required** | Target potential energy |
| `FORCE_KEY` | `"forces"` | `torch.float32` | **Required** | Target atomic forces |
| `PBC_KEY` | `"pbc"` | `torch.bool` | **Required** | Boolean flags `[3]` for PBC |
| `CELL_KEY` | `"cell"` | `torch.float32` | **Required** | Lattice cell matrix `[3, 3]` |
| `EDGE_INDEX_KEY` | `"edge_index"` | `torch.long` | Optional | Neighbor connectivity |

---

## 🔍 2. Identified Compatibility Gaps

1. **Atomic Numbers Key Mismatch**:
   - *Gap*: The preprocessing pipeline saves the atomic numbers list under the key `z` (and PyG's `x` attribute).
   - *NequIP Requirement*: NequIP explicitly searches for `atomic_numbers` (`AtomicDataDict.ATOMIC_NUMBERS_KEY`).
   - *Impact*: Incompatible unless a manual configuration mapping is added via `key_mapping`.

2. **Energy Key Mismatch**:
   - *Gap*: The pipeline saves the total energy under the key `y` (the standard PyTorch Geometric target name).
   - *NequIP Requirement*: NequIP uses `total_energy` (`AtomicDataDict.TOTAL_ENERGY_KEY`) for regression.
   - *Impact*: Leads to training failures/KeyErrors since NequIP models default to `total_energy`.

3. **Missing PBC and Cell Tensors**:
   - *Gap*: The current script does not export `pbc` or `cell` tensors.
   - *NequIP Requirement*: In NequIP version 0.3.0+, `pbc` (shape `[3]`, boolean) and `cell` (shape `[3, 3]`, float32) must be defined to prevent ambiguity in periodic image and neighbor-list resolution.
   - *Impact*: Potential KeyErrors or failures when constructing neighbor lists or when loading graphs into `AtomicData`.

---

## 🛠️ 3. Required Fixes in Preprocessing Pipeline

To resolve all compatibility gaps without breaking backward compatibility for other graph neural network frameworks (e.g., EGNN or standard PyG scripts), the following updates must be made to `scripts/preprocess.py`:

- **Atomic Numbers**: Save the atomic numbers under **both** `z` and `atomic_numbers`.
- **Total Energy**: Save the energy target under **both** `y` and `total_energy`.
- **Periodic Boundary Conditions**: Add default tensors:
  - `pbc` = `torch.tensor([False, False, False], dtype=torch.bool)`
  - `cell` = `torch.zeros((3, 3), dtype=torch.float32)`
  This represents non-periodic isolated systems (such as molecular dynamics trajectories of single molecules).
